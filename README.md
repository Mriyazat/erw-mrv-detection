# ERW MRV Detection Limits

Reproducible analysis pipeline for a study of the **measurement, reporting and
verification (MRV) limits of enhanced rock weathering (ERW)** at a fully
instrumented field trial. The code supports two companion manuscripts that
reach a common conclusion from different directions: at realistic field
replication, the ERW signal sits near the detection floor, and verification is
limited by soil heterogeneity rather than by the choice of measurement phase or
statistical model.

## Associated manuscripts

1. *Enhanced rock weathering verification is detection-limited across
   measurement phases: an aqueous-phase detection budget from a 12-plot
   wollastonite–diopside field trial* — an aqueous/exchange-phase detection
   budget, its convergence with the published solid-phase heterogeneity wall,
   and a feedstock-attributed cation-exchange "CDR lag."
2. *Forecastable but not detectable: a signal-to-noise and leakage-controlled
   machine-learning study of enhanced rock weathering from soil sensors* — a
   first-principles signal-to-noise budget, a leakage-controlled machine-learning
   analysis (the *Detection–SNR consistency criterion*), and an honest
   foundation-model forecasting benchmark.

This repository contains the code and pipeline behind both manuscripts.

## The field trial

- **Site:** research station on an Ontario clay-loam (humid continental).
- **Design:** twelve instrumented plot-halves; a 50:50 wollastonite–diopside
  amendment applied at 0 / 20 / 60 t ha⁻¹ (N = 4 plot-halves per arm).
- **Measurement streams:**
  1. *TEROS* soil sensors — 15-minute volumetric water content, temperature,
     bulk electrical conductivity and matric potential at 15 / 40 / 100 cm,
     spanning a continuous May 2025 – April 2026 year (~3.9 × 10⁵ records).
  2. *UNIBEST PRS* ion-exchange resin capsules — three deployment rounds,
     three depths, sixteen analytes.
  3. Co-located 15-minute weather station.
  4. *eosAC* autochamber gas-flux campaign (analysed separately).

## Repository layout

```
src/         configuration, data loaders, physics (SNR model), statistics,
             machine-learning utilities (leakage-safe CV, conformal UQ), and
             effect-size / detection-budget analysis code
scripts/     numbered, self-contained analysis stages (00–116); each writes
             result tables (CSV), audit notes (Markdown) and figures
tests/       golden-master and extraction tests
Makefile     convenience targets for each analysis stage
```

## Reproducing the analysis

The pipeline is deterministic (fixed random seed) and organised as a numbered
sequence of stages.

```bash
# 1. Environment
pip install -r requirements.txt          # or: conda env create -f environment.yml

# 2. Sanity check (imports, configuration, optional dependencies)
make smoke

# 3. Build the processed data caches from the raw instrument files
make data

# 4. Run individual analysis stages, e.g.
make snr            # first-principles signal-to-noise model
make power          # detection budget / minimum detectable effect
make test           # golden-master + extraction tests
```

Each `scripts/NN_*.py` stage can also be run directly with
`python scripts/NN_*.py`. Stage outputs are written under `outputs/`
(git-ignored; regenerated by the pipeline). The most computationally intensive
machine-learning experiments (deep and foundation time-series models) are
intended for a GPU node; see `scripts/hpc/`.

## Data availability

Raw instrument files (sensor, resin, weather and chamber workbooks) are not
redistributed in this repository because of their size and access conditions.
The pipeline expects them under `data/`; processed Parquet caches are generated
by `make data`. Researchers seeking the raw data should contact the trial
operators. All derived result tables and figures are fully reproducible from the
raw data with the code provided here.

## Software components

The analysis uses NumPy, pandas and scikit-learn throughout; statsmodels for
power analysis; NumPyro/JAX for the hierarchical Bayesian model; LightGBM,
XGBoost and CatBoost for the gradient-boosting baselines; and transformer
foundation time-series models (Chronos, TimesFM, Moirai) used zero-shot for the
forecasting benchmark. Uncertainty is quantified with distribution-free
conformal prediction. See `requirements.txt` for pinned versions.
